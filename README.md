ОПИСАНИЕ:

Новостной агрегатор в формате RSS читалки для телеграм каналов.

В основе лежит парсинг данных из репозиторий https://t.me.
Очевидное преимуществом перед обычным чтением из приложения - это отсутствие авторизации, трэкеров аналитики.
https://t.me также можно читать без авторизации через любой браузер. Но и здесь преимущество приложения - это легковесность. 
Это достигается за счет библиотеки JSoup. Загружается только текстовый формат веб страницы без UI элементов и конечно же без джава скриптов.


Загрузив канал через приложение можно читать его оффлайн.
Для кого-то недостатком может показаться отсутствие видео, аудио, картинок. Формат приложения RSS все таки подразумевает только текстовую информацию. 
Хотя при желании можно и url-ы картинок распарсить и показывать в постах.

Приложение назвал агрегатором, т.к. в дальнейшем посты будут вытягиваться из разных добавленных пользователем каналов и перемешиваться в ленте.

БИЗНЕС ЛОГИКА:

Приложение вытягивает все данные из официального https://t.me. (В перспективе можно добавить tgstat.com).
Пользователь вводит имя канала и приложение начинает парсинг с базового url https://t.me/s/{имя_канала}. Из этого запроса получаю только общее количество постов на канале с начала его основания.
Далее все посты, начиная с крайнего, вытягиваю через отдельный GET запрос для каждого поста, в порядке убывания номера поста. Для этого использую https://t.me/{имя_канала}/{номер_поста}.


АРХИТЕКТУРА:

Приложение работает по принципам Clean Architecture, описанным в мануалах developer.android.com. Использую паттерн Model-View-ViewModel.
Вводил только 2 слоя : Ui и Data. Домен был бы излишним. 

Слой UI включает в себя host activity с фрагментами (mvvm). А также пакет ui.paging с адаптером, пейджером и viewholder для отображения списка.

Слой Data включает в себя PagingSource для работы с пагинацией. Репозиторий, реальзующий интерфейс и работающий с ViewModel. А также DataSource, который вытягивает данные через корутин диспатчеры из Jsoup api.

Из внешних библиотек использовал JSoup parser. В библиотеке есть встроенный http клиент, поэтому Retrofit не использовался.
Асинхронность запросов Jsoup GET делал через корутины в связке с lifecycle scope.
Для внедрения зависимостей использовал hilt.

Для пагинации использовал Paging3 библиотеку. Библиотека очень хорошо разграничивает Paging api в 2 слоях ui и data.
Для навигации между компонентами использовал Navigation API с популярным сейчас принципом Single app activity.
